Problem setup:

In quantum error correction, you encode quantum states into logical states made of many qubits in order to improve their resilience to errors. In quantum error detection, you do the same but can only detect the presence of errors and not correct them. In this problem, we will consider a single [[4,2,2]] quantum error detection code, which encodes two logical qubits into four physical qubits, and investigate how robust logical quantum operations in this code are to quantum errors.

Our convention is that the four physical qubits in the [[4,2,2]] code are labelled 0,1,2,3. The two logical qubits are labelled A and B. The stabilizers are \(XXXX\) and \(ZZZZ\), where \(X\) and \(Z\) are Pauli matrices. The logical \(X\) and \(Z\) operators on the two qubits are \(X_A = XIXI\), \(X_B=XXII\), \(Z_A = ZZII\), \(Z_B = ZIZI\), up to multiplication by stabilizers.

We will consider different state preparation circuits consisting of controlled not \(CNOT_{ij}\) gates, where \(CNOT_{ij}\) has control qubit \(i\) and target qubit \(j\). As a simple model of quantum errors in hardware, we will suppose that each \(CNOT_{ij}\) gate in the circuit has a two qubit depolarizing error channel following it that produces one of the 15 non-identity two-qubit Paulis with equal probability \(p/15\). The probability \(p\) indicates the probability of an error in a single two-qubit gate. We will assess the logical infidelity of certain state preparation protocols as a function of the physical infidelity \(p\).

Challenge

Suppose that we prepare a logical two-qubit \(|00\rangle_{AB}\) state in the [[4,2,2]] code. To do so, we introduce an ancilla qubit, qubit 4, and use the following state preparation circuit:

\[ M_4 (CNOT_{04}) (CNOT_{34}) (CNOT_{23}) (CNOT_{10}) (CNOT_{12}) (H_1) \]

Note that this equation is written in matrix multiplication order, while the quantum operations in the circuit occur in the reverse order (from right-to-left in the above equation). \(H\) is a single-qubit Hadamard gate and \(M\) is a single-qubit measurement. The ancilla is used to detect errors in the state preparation circuit and makes the circuit fault-tolerant. If the ancilla measurement is \(|0\rangle\) (\(|1\rangle\)), the state preparation succeeds (fails).

What is the logical state fidelity of the final 2-qubit logical state at the end of the circuit as a function of two-qubit gate error rate \(p\), assuming the state is post-selected on all detectable errors in the code and on the ancilla qubit measuring \(|0\rangle\)?

# Represent n-qubit Paulis with (x,z) bit vectors in GF(2) ignoring global phase.
# Qubit order: 0,1,2,3,4 (4 = ancilla)

nq = 5

# CNOT conjugation mapping
# For CNOT with control c and target t, mapping on (x,z):
# x_c' = x_c
# x_t' = x_t XOR x_c
# z_c' = z_c XOR z_t
# z_t' = z_t

def conj_cnot(x,z,c,t):
    x = x.copy(); z = z.copy()
    # Save old values
    xc, xt = x[c], x[t]
    zc, zt = z[c], z[t]
    # Apply mapping
    x[t] = xt ^ xc
    # x[c] unchanged
    z[c] = zc ^ zt
    # z[t] unchanged
    return x, z

# Compose two Paulis P1*P2 ignoring phase: XOR of x and z bits

def compose(p1, p2):
    x1,z1 = p1; x2,z2 = p2
    return [ [a^b for a,b in zip(x1,x2)], [a^b for a,b in zip(z1,z2)] ]

# Generate 1- and 2-qubit Pauli on specified qubits (qa, qb) with given single-qubit pauli labels 'I','X','Y','Z'
label_to_xz = {
    'I': (0,0),
    'X': (1,0),
    'Z': (0,1),
    'Y': (1,1),
}

paulis_1q = ['I','X','Y','Z']

# Build a 5-qubit Pauli from per-qubit labels

def build_pauli_from_labels(labels):
    x = [0]*nq
    z = [0]*nq
    for i,L in enumerate(labels):
        xi,zi = label_to_xz[L]
        x[i]=xi; z[i]=zi
    return [x,z]

# Convenience: build a 2-qubit Pauli on qubits (a,b), identity elsewhere

def pauli_on_pair(a,b,La,Lb):
    labels=['I']*nq
    labels[a]=La; labels[b]=Lb
    return build_pauli_from_labels(labels)

def symp_commutes(p, q):

# Project to 4-qubit data vector (drop ancilla bits)

# Linear algebra over GF(2) to decompose E_data = a XA + b XB + c ZA + d ZB + alpha S1 + beta S2
# Return tuple (a,b,c,d,alpha,beta) in GF(2), or None if E_data not in centralizer (shouldn't call in that case)

def decompose_to_logicals(E_data):
    # Build matrix with columns for [XA, XB, ZA, ZB, S1, S2]
    basis = [XA, XB, ZA, ZB, S1, S2]
    # We'll work with 8 rows (x0..x3, z0..z3)
    import numpy as np
    M = np.zeros((8,6), dtype=int)
    for j,B in enumerate(basis):
        xb,zb = drop_ancilla(B)
        M[0:4,j] = xb
        M[4:8,j] = zb
    # RHS vector v
    xv,zv = E_data
    v = np.array(xv + zv, dtype=int)
    # Solve M c = v over GF(2)
    # Use Gaussian elimination
    A = np.concatenate([M, v.reshape(-1,1)], axis=1)
    m,n = A.shape
    # Row-reduce
    row = 0
    pivots = []
    for col in range(6):
        # find pivot
        pivot = None
        for r in range(row, m):
            if A[r,col] % 2 == 1:
                pivot = r; break
        if pivot is None:
            continue
        # swap
        if pivot != row:
            A[[row,pivot]] = A[[pivot,row]]
        pivots.append(col)
        # eliminate other rows
        for r in range(m):
            if r != row and (A[r,col] % 2 == 1):
                A[r,:] ^= A[row,:]
        row += 1
        if row == m:
            break
    # Check consistency: all-zero rows on left must have zero RHS
    for r in range(row, m):
        if A[r,0:6].sum() % 2 == 0:
            if A[r,6] % 2 == 1:
                return None
    # Extract a solution by back substitution: pick zeros for free vars
    sol = np.zeros(6, dtype=int)
    # Starting from bottom rows, set pivot variable to RHS - sum(free*coeffs)
    # Here, above elimination has made columns isolated; we can read them
    for i, col in enumerate(pivots):
        # Find the row where this pivot is 1
        r = None
        for rr in range(m):
            if A[rr,col] == 1:
                r = rr; break
        rhs = A[r,6]
        # subtract contributions of already set vars
        for c2 in range(6):
            if c2 == col: continue
            if A[r,c2] == 1:
                rhs ^= sol[c2]
        sol[col] = rhs
    a,b,c,d,alpha,beta = [int(x)%2 for x in sol.tolist()]
    return a,b,c,d,alpha,beta

We post-select on: - ancilla measurement \(|0\rangle\); and - “all detectable code errors,” i.e., errors that would flip either stabilizer \(XXXX\) or \(ZZZZ\) (equivalently, errors that anticommute with at least one stabilizer) are rejected.
